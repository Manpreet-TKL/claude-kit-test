# OpenEyes runtime components

When loaded as context with no task, reply only `Context loaded.` This skill is context-only: it never does anything by itself — it just loads knowledge; act only on instructions given in the conversation.

What executes inside a running deployment — `c-oeimagebuilder` builds the images, `c-oe-deploy` assembles and runs them. Detail: `subs/cron.md` (master-container yiic schedule), `subs/env-vars.md` (expected env vars), `subs/tmp-artefacts.md` (which rendering/import tool creates which temp files, and where each is invoked).

## Containers

| Container | Image | Runs |
|---|---|---|
| `web` | `oe-web-live:<tag>` | Apache 2.4 (mpm-event) + PHP-FPM 8.4; serves all HTTP |
| `oe-manager` | `oe-manager:<tag>` | Same stack `FROM oe-web-live`; admin/offloaded ops. **Tag must match `web`** |
| `master` | `oe-master:<tag>` | Cron — the scheduled `yiic` commands |
| `db` | `mariadb:<pinned>` | Named volume `db_data`; `my.cnf` bind-mounted |
| `redis` | `redis:<pinned>` | Queue backend + cache |
| `traefik` | `traefik:<pinned>` | HTTP reverse proxy |
| `bridgelink` | `nextgenhealthcare/bridgelink:<patch>` | Mirth fork — pin a patch (`4.6.1`); bare `4.6` doesn't exist |

## Inside web

Apache → `index.php` (only entry); APCu (config cache `oe_merged_config_*`); TCPDF/FPDF/FPDI (letters, CVI certs); Puppeteer via `DocumentRenderServicePuppeteer` (HTML→PDF, Chromium ships in the image); TinyMCE (`params.tinymce_default_options`); Eyedraw; optional ClamAV when `OE_ENABLE_VIRUS_SCANNING=true`; SAML/OAuth2/OIDC (`params.SAML_settings` / `params.OIDC_settings`); Vite-built TS/Tailwind assets.

## Queues and storage

- `asyncJobDispatcher` → `database` queue unless `QUEUE_CONNECTION=redis`; Laravel Horizon (`./oe-laravel/artisan horizon`) runs against Redis when enabled; Yii batch jobs run via master cron.
- Bind mounts survive upgrades: `event_images/`, `protected/files/`, `DicomFiles`, `event_export_location` (NOD/analytics). `FileStorage` module = pluggable local-FS/S3 backend.
- `cachebuster.txt` in `protected/config/` (not git-tracked) — asset-URL cache key, regenerated by deploy.
- DB creds resolution order: `/etc/openeyes/db.conf` → `DATABASE_*` env → `/run/secrets/DATABASE_*` (same for `$db_test`). `OE_COOKIE_VALIDATION_KEY` is required (env or secret).

## Host notes

- monkey: Traefik HTTP-only on `:80`, PathPrefix routing — `/` → OE, `/openmrs/spa` → SPA (strip), `/openmrs` → backend; no DNS, no TLS.
- Traefik < v3.6 needs the Docker daemon's `min-api-version` lowered to `1.24` in `/etc/docker/daemon.json` — fix the daemon, don't swap the Traefik tag or add a socket proxy.
